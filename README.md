# Feature Voting Board

This is a simple project built with HTML, CSS, and JavaScript. It allows users to see a list of features, upvote them, and add their own suggested features.

## Features
- Upvote buttons that increase the vote count.
- Real-time ranking (the list re-orders itself automatically).
- An input box to add your own feature requests.
- LocalStorage so the data stays there even if you refresh the page.

## PM Note: Which feature to ship first?

If I were the Product Manager, I would ship **Dark Theme** first, regardless of the votes.

### Why?
1. **Low Effort:** Dark Theme is mostly a design change in the CSS. It is much faster to build than complex features like "Video Calls" or "Voice Messages."
2. **User Comfort:** It is a very popular request for accessibility and helps users who prefer using the app in low-light environments.
3. **Speed to Market:** It allows the team to deliver a "win" to the users quickly while the developers take more time to work on the bigger, more difficult features.

## How to Run
Just open the `index.html` file in any web browser.
